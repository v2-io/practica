# s3-working — incremental per-chapter notes on Source 3

Working substrate for the third 02-harvest input — a book Joseph supplies one chapter at a time. Distinct directory because the *protocol* is different from the AAT/operata harvest: incremental, no peeking, no multi-file reads. The point of the protocol is the discipline; the file structure is its durable artifact across context turnover.

## Protocol (Joseph, 2026-05-19)

- **One chapter at a time.** Joseph supplies; I read; I write the file; he supplies the next.
- **No multi-file read. No peeking ahead.** Even if I recognize the book.
- **The catch.** Joseph already knows this book is (a) well represented in my LLM training and (b) mentioned in both AAT and operata literature. The point is the *incremental* process — having only this chapter attendable instead of an omniscient training-broad-strokes view of the whole. Surprise and consequence often surface in this protocol that don't from the omniscient pose.
- **Discipline that follows:** write *from this chapter*, not from training. If something is a training-pattern-match rather than chapter-induced, name it as such honestly in §3. Predictions in §2 are predictions, not training-recall.

## Per-chapter file (`chNN.md`, zero-padded)

Four sections, exactly:

1. **Critical concepts, reasonings, & conclusions.**
2. **Updated predictions about key things the rest of the book will contain.** (First-pass in `ch01.md`; updated thereafter, with what *this* chapter changed about the prediction.)
3. **Diffuse Wandering Mind.** 3–10 paragraphs of tangents, implications, call-backs, tie-ins, questions, ideas triggered by what's *new* in this chapter. Legitimate channel for spontaneous association; honest about training-pattern-match where it shows up.
4. **Applicability to Practica.** Directly, indirectly, or possibly applicable. Consolidates back into [[../02-normative-harvest]] later, by provenance tag `src3`.
